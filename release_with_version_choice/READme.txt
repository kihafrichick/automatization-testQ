./autotest_qaz:

	./lib —————————————————————————————————————— каталог для библиотек и классов
		./KIWItcms ————————————————————————— каталог для работы с KIWI
			./kiwitcmsclient.py ———————— клиент для работы с KIWI
		./__init__.py —————————————————————— файл необходим для определения каталога как пакета
		./UtilsManager.py —————————————————— класс с различными утилитами, такие как загрузка конф.файла, и маленькая обвязка работы с KIWIClient
	
	./web —————————————————————————————————————— каталог для тестов веб-интерфейса
		./perl1_start_script.py ———————————— выполняет рестарт передачи данных на анализаторе и выводит текущий статус соединения. Связан с RC-5800.pl
		./pre-final_vers.py ———————————————— управляющий скрипт, объединяющий работу всех скриптов в каталоге
		./RC-5800.pl ——————————————————————— данный скрипт обеспечивает взаимодействие с сетевым анализатором Viavi MTS-5800 через SCPI запросы
		./sborka_sinhron.py ———————————————— данный скрипт содержит модули, которые выполняют проверку функциональности веб-интерфейса для 1-го устройства из пары
		./sborka_sinhron_blade.py —————————— данный скрипт содержит модули, которые выполняют проверку функциональности веб-интерфейса для 2-го устройства из пары
	
	./make_test_run.py ————————————————————————— скрипт, создает в КIWI тест прогон из тест плана, указанного в конфигурационном файле, и добавляет туда все тест кейсы из тест-плана
	./set_conf.json ———————————————————————————— файл, в который записывается выбранный для работы конфиг-файл
	./config_vX-X-XX.json —————————————————————— файл конфигурации,содержащий параметры,необходимые для тестирования конкретной версии системы управления "Атлас" на Квазаре
	./autorun.py ——————————————————————————————— скрипт, автоматически запускает вначале make_test_run, затем pre-final_vers.py

*-./__pycache__ эти каталоги содержат скомпилированные версии файлов исходного кода.Создаются автоматически для ускорения выполнения скриптов. Появятся после 1-го запуска.**

**-также, после первого запуска, в директории /web появятся по два файла info_xxx.txt и по два файла output_xxx.txt. xxx - номера устройств соответствующие значениям "web_ip" и "web_ip_blade" из конфиг-файла(об этом ниже). info_xxx.txt содержит информацию из таблицы со вкладки "Монитор", а output_xxx.txt содержит актуальные записи из журнала "Квазар".

Как запускать автотест:

	а). Для запуска скрипта на полный цикл тестирования необходимо перейти в каталог autotest_qaz и в терминале ввести команду: <python3 ./autorun.py> 
	б). Для запуска отдельных тестовых групп необходимо ввести ту же команду с добавлением опции: <python3 ./autotest_qaz --groups=1,2,3-7>
	   Нумерация групп производится в соответствии с порядком их выполнения,ниже представлен перечень групп и функции,которые они выполняют:
	   
	   1 -- Инициализация и аутентификация в качестве админа СУ "Атлас"
	   2 -- Считывание информации из таблицы вкладки "Монитор" 
	   3 -- Включение первой,второй и клиентской линий на проверяемых устройствах
	   4 -- Установка статуса "Управляется ALS" в настройках передатчика SFP+ линейных портов 1/2/Cl
	   5 -- Переключение FEC на первой и второй линиях в режим "EFEC" 
	   6 -- Переключение FEC на первой и второй линиях в режим "G.709"
	   7 -- Переключение FEC на первой и второй линиях в режим "Выкл."
	   8 -- Переключение FEC на первой и второй линиях обратно в режим "G.709" для проверки статусов АLS
	   9 -- Переключение ALS в режимы "Перезапуск автоматический" для первого устройства и "Перезапуск импульсный" для второго
	   10 -- Переключение ALS в режимы "Перезапуск импульсный" для первого устройства и "Перезапуск автоматический" для второго
	   11 -- Переключение ALS в режимы "Выкл" для обоих устройств.
	   12 -- Выключение первой,второй и клиентской линий на проверяемых устройствах
	   13 -- Считывание информации из журнала "Квазар",доступного по ссылке-названию в перечне устройств на главной странице СУ "Атлас"
	   
	в). Для запуска скрипта с помощью pytest необходимо ввести команду <python3 ./autorun.py --pytest>
	г). Также есть опция help <-h, --help>, запустив скрипт с этой опцией вы узнаете в каком порядке должны быть переданы опции и какие есть варианты их передачи.
	Вот содержание после запуска команды с опцией help:
		
	usage: pre-final_vers.py [-h/--help] [--firefox/fx] [--groups=GROUPS] [--pytest] [--headless]
	
	Памятка по работе с autorun.py:
	
	Опции:
  	-h, --help            Показать это сообщение и выйти
  	--groups , -g 	      Номера групп для запуска (например: groups=1,3,5-8)                       
  	--pytest              Запустить через pytest
    --no-kiwi             Запустить без создания test run в Kiwi TCMS
    --headless            Опция для запуска без отображения действия скрипта
    --firefox/fx          Выбор браузера Firefox (по умолчанию скрипт запустится в Google-chrome)



	д). Далее, после запуска скрипта появится строчка "Выберите файл 1-Х или 0 для выхода:", где 1-Х перечень номеров доступных файлов конфигураций. 
	Сравните текущую версии СУ "Атлас" с версиями доступных файлов конфигураций и выберите нужный.
